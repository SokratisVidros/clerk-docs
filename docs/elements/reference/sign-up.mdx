# Clerk Elements sign-up components

The following components are used when creating sign-up flows. They are imported from `@clerk/elements/sign-up`. We recommend importing them all under the `SignUp.` namespace to make discovery easier and reduce naming conflicts with other components throughout your application. The code snippets on this page assume you have imported the components this way.

```tsx filename="Anatomy"
<SignUp.Root>
  <SignUp.Step name="start">
    <SignUp.Action submit>
  <SignUp.Step>
  <SignUp.Step name="continue">
    <SignUp.Action submit>
  </SignUp.Step>
  <SignUp.Step name="verifications">
    <SignUp.Strategy name="email_code">
      <SignUp.Action submit>
    </SignUp.Strategy>
  </SignUp.Step>
</SignUp.Root>
```

## `<Root>`

The root sign-up component. Sets up providers and state management for the sign-up flow. Must wrap all other sign-up components. `<Root>` will validate your built sign-up flow to ensure the implementation is correct based on instance settings and best practices.

### Properties

| Name | Type                | Description                                                       |
| ---- | ------------------- | ----------------------------------------------------------------- |
| `path` | `string` (optional) | The root path the sign-up flow is mounted at. Default: `/sign-up` |
| fallback | `React.ReactNode` (optional) | Fallback markup to render while Clerk is loading. |

## `<Step>`

A step in the sign-up flow. The component controls conditionally rendering its children based on the status of the current sign up attempt. `start` is the initial step.

### Properties

| Name | Type     | Description                                                     |
| ---- | -------- | --------------------------------------------------------------- |
| `name` | `start \| continue \| verfications` | The `name` of the step for which its children will be rendered. |

### `<Step name="start">`

Used to render the beginning sign-up form. Once a sign up attempt has been created from this step, the `continue` or `verification` step will be rendered. The exact fields that should be rendered depends on your instance configuration.

#### Example

```tsx
<SignUp.Step name="start">
  <Clerk.Provider name="google">Sign up with Google</Clerk.Provider>

  <Clerk.Field name="identifier">
    <Clerk.Label>Email</Clerk.Label>
    <Clerk.Input />
    <Clerk.FieldError />
  </Clerk.Field>

  <SignUp.Action submit>Sign up</SignUp.Action>
</SignUp.Step>
```

### `<Step name="continue">`

Used to collect additional required fields from the user during a sign up attempt. This step will be rendered if a user initiates a sign up via a social provider, but that provider does not provide all required fields.

#### Example

```tsx
<SignUp.Step name="continue">
  <Clerk.Field name="username">
    <Clerk.Label>Username</Clerk.Label>
    <Clerk.Input />
    <Clerk.FieldError />
  </Clerk.Field>

  <SignUp.Action submit>Sign up</SignUp.Action>
</SignUp.Step>
```

### `<Step name="verifications">`

Used to verify certain fields provided during sign up. If your instance is configured to require verification of emails or phone numbers, this step will render.

#### Example

```tsx
<SignUp.Step name="verifications">
  <SignUp.Strategy name="email_code">
    <Clerk.Field name="code">
      <Clerk.Label>Email code</Clerk.Label>
      <Clerk.Input />
      <Clerk.FieldError />
    </Clerk.Field>

    <SignUp.Action submit>Verify email</SignUp.Action>
  </SignUp.Strategy>
</SignUp.Step>
```

## `<Strategy>`

Used to conditionally render its children depending on the authentication strategy that needs to be verified. Does not render any markup on its own.

### Properties

| Name | Type     | Description                                                       |
| ---- | -------- | ----------------------------------------------------------------- |
| `name` | `string` | The name of the strategy for which its children will be rendered. |

### Examples

#### Basic

```tsx
<SignUp.Strategy name="email_code">
  <Clerk.Field name="code">
    <Clerk.Label>Code</Clerk.Label>
    <Clerk.Input />
    <Clerk.FieldError />
  </Clerk.Field>

  <SignUp.Action submit>Verify</SignUp.Action>
</SignUp.Strategy>
```

## `<Action>`

The `<Action>` component exposes various flow-related actions. It can be used to submit forms, navigate between steps, and re-trigger sending of verification codes. By default, renders a `<button>`.

### Properties

| Name       | Type                 | Description                                                                                               |
| ---------- | -------------------- | --------------------------------------------------------------------------------------------------------- |
| `submit`     | `boolean` (optional) | If `true`, the action will submit the form.                                                               |
| `navigate` | `start` \| `previous` (optional)  | The name of the step to navigate to.                                                                      |
| `resend`   | `boolean` (optional) | If `true`, the action will resend the verification code for the currently active strategy, if applicable. |
| `fallback` | `({ resendableAfter: number }) => React.ReactNode` (optional) | Only used when `resend` is `true`. If provided, the fallback markup will be rendered before the resend delay has expired. |

### Examples

#### Submit

```tsx {8}
<SignUp.Step name="start">
  <Clerk.Field name="identifier">
    <Clerk.Label>Email</Clerk.Label>
    <Clerk.Input />
    <Clerk.FieldError />
  </Clerk.Field>

  <SignUp.Action submit>Sign up</SignUp.Action>
</SignUp.Step>
```

#### Navigate

```tsx {9}
<SignUp.Step name="continue">
  <Clerk.Field name="username">
    <Clerk.Label>Username</Clerk.Label>
    <Clerk.Input />
    <Clerk.FieldError />
  </Clerk.Field>

  <SignUp.Action submit>Sign up</SignUp.Action>
  <SignUp.Action navigate="start">Go back</SignUp.Action>
</SignUp.Step>
```

#### Resend

```tsx {10-15}
<SignUp.Step name="verifications">
  <SignUp.Strategy name="email_code">
    <Clerk.Field name="code">
      <Clerk.Label>Code</Clerk.Label>
      <Clerk.Input />
      <Clerk.FieldError />
    </Clerk.Field>

    <SignUp.Action submit>Verify</SignUp.Action>
    <SignUp.Action
      resend
      fallback={({ resendableAfter }) => <p>Resend code in {resendableAfter} second(s)</p>}
    >
      Resend code
    </SignUp.Action>
  </SignUp.Strategy>
</SignUp.Step>
```
