# Clerk Elements common components

Clerk Elements provides a set of common components that are used across all flows. We recommend importing them all under the `Clerk.` namespace to make discovery easier and reduce naming conflicts with other common components throughout your application. The code snippets on this page assume you have imported the components this way.

```tsx
import * as Clerk from '@clerk/elements/common'
```

Unless otherwise mentioned, all components accept a `className` prop.

## `<Field>`

Field is used to associate its child elements with a specific input. It automatically handles unique ID generation and associating the contained label and input elements.

### Props

| Name       | Type                                                          | Description                                                                      |
| ---------- | ------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| `name`     | `string`                                                      | Name for form field, will be used to associate all underlying field elements.    |
| `children` | `(state: 'valid' \| 'invalid') => React.ReactNode` (optional) | Optionally, you can provide a function as children and access the field's state. |

### Examples

#### Basic

```tsx
<Clerk.Field name="unique-name">{/* Your content */}</Clerk.Field>
```

#### Function as children

```tsx
<Clerk.Field name="unique-name">{(state) => <p>Field's state is: {state}</p>}</Clerk.Field>
```

## `<FieldError>`

FieldError renders error messages associated with a specific field. By default, the error's message will be rendered in a `span`. Optionally, the `children` prop accepts a function to completely customize rendering.

### Properties

| Name       | Type                                                                | Description                                                                                 |
| ---------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| `children` | `({ message: string, code: string }) => React.ReactNode` (optional) | Optionally, you can provide a function as children and access the error's message and code. |

### Examples

#### Basic

```tsx
<Clerk.Field name="unique-name">
  <Clerk.FieldError />
</Clerk.Field>
```

#### Function as children

```tsx
<Clerk.Field name="unique-name">
  <Clerk.FieldError>
    {({ message, code }) => (
      <span>
        {message} ({code})
      </span>
    )}
  </Clerk.FieldError>
</Clerk.Field>
```

## `<FieldState>`

Programmatically access the state of the wrapping `<Field>`. Useful for implementing animations when direct access to the state value is necessary.

### Properties

| Name       | Type                                               | Description                                         |
| ---------- | -------------------------------------------------- | --------------------------------------------------- |
| `children` | `(state: 'valid' \| 'invalid') => React.ReactNode` | Use this function prop to access the field's state. |

### Examples

#### Basic

```tsx
<Clerk.Field name="unique-name">
  <Clerk.FieldState>{(state) => <p>Field's state is: {state}</p>}</Clerk.FieldState>
</Clerk.Field>
```

## `<GlobalError />`

Used to render errors that are returned from the API, but not associated with a specific form field. By default, will render the error's message wrapped in a `div`. Optionally, the `children` prop accepts a function to completely customize rendering. Must be placed **inside** components like `<SignIn>`/`<SignUp>` to have access to the underlying form state.

### Properties

| Name       | Type                                                                | Description                                                                                 |
| ---------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------- |
| `children` | `({ message: string, code: string }) => React.ReactNode` (optional) | Optionally, you can provide a function as children and access the error's message and code. |

### Examples

#### Basic

```tsx
<SignIn.Root>
  <Clerk.GlobalError />
</SignIn.Root>
```

#### Function as children

```tsx
<SignIn.Root>
  <Clerk.GlobalError>
    {({ message, code }) => (
      <span>
        {message} ({code})
      </span>
    )}
  </Clerk.GlobalError>
</SignIn.Root>
```

## `<Label>`

Renders a `<label>` element that is automatically associated with its sibling `<Input />` inside of a `<Field>`. Additional props will be passed through to the [`<label>` element](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/label).

### Properties

| Name    | Type                 | Description                                                                               |
| ------- | -------------------- | ----------------------------------------------------------------------------------------- |
| asChild | `boolean` (optional) | If true, `<Label />` will render as its child element, passing along any necessary props. |

### Examples

#### Basic

```tsx
<Clerk.Field name="unique-name">
  <Clerk.Label>Label</Clerk.Label>
</Clerk.Field>
```

#### With `asChild`

```tsx
<Clerk.Field name="unique-name">
  <Clerk.Label asChild>
    <label className="my-custom-label">Label</label>
  </Clerk.Label>
</Clerk.Field>
```

## `<Input>`

Handles rendering of `<input>` elements within Clerk's flows. Supports multiple special `type` prop values to render input types that are unique to authentication and user management flows. Additional props will be passed through to the [`<input>` element](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/input).

### Properties

| Name    | Type                                                  | Description                                                                                                                                           |
| ------- | ----------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------- |
| asChild | `boolean` (optional)                                  | If true, `<Input />` will render as its child element, passing along any necessary props.                                                             |
| name    | `string` (optional)                                   | Name for form field, will be used to associate all underlying field elements. Can be automatically inferred from the name on the `<Field>` component. |
| type    | `text \| email \| tel \| otp* \| password` (optional) | Type for the input. Default: `text`                                                                                                                   |

_Values denoted with \* are unique Clerk input types._

The following data attributes are also added to the underlying Element:

- `data-valid` - If the field state is valid
- `data-invalid` - If the field state is invalid (has an associated error)

### Examples

#### Basic

```tsx
<Clerk.Field name="identifier">
  <Clerk.Label>Email</Label>
  <Clerk.Input type="email" />
</Clerk.Field>
```

## `<Input type="otp">`

A special type used to render an input that accepts a one-time passcode (OTP). If the corresponding `Field` has `name="code"`, the contained `Input` will default to the `otp` type. Only numbers are accepted as inputs, and the default max length is 6.

By default, a single text `<input />` will be rendered. If provided, the `render` prop will be called for each character currently present in the input. This enables UI patterns where an OTP input is visually represented as N distinct elements.

### Properties

| Name                  | Type                 | Description                                                                                                                                                                                                                                                                                   |
| --------------------- | -------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| length                | `number` (optional)  | Define how many numbers the input should hold. Default: 6                                                                                                                                                                                                                                     |
| autoSubmit            | `boolean` (optional) | If true, the form will be automatically submitted once the input is filled out. Default: false                                                                                                                                                                                                |
| passwordManagerOffset | `number` (optional)  | Password managers place their icon inside an `<input />`. This default behaviour is not desirable when you use the render prop to display N distinct element. With this prop you can increase the width of the `<input />` so that the icon is rendered outside the OTP inputs. Default: `40` |
| `render`              | `({ value, status }: { value: string; status: 'none' \| 'selected' \| 'cursor' \| 'hovered' }) => React.ReactNode` (optional) | A render prop function that receives the value and status of each character. value is the character to render, status is the current status of the input character.                                                                                                                           |

### Examples

#### Basic

```tsx
<Clerk.Field name="code">
  <Clerk.Input type="otp" autoSubmit />
</Clerk.Field>
```

#### Segmented

```tsx
<Clerk.Field name="code">
  <Clerk.Input
    type="otp"
    passwordManagerOffset={40}
    render={({ value, status }) => <span data-status={status}>{value}</span>}
  />
</Clerk.Field>
```

## `<Loading>`

Access the loading state of a chosen scope. Scope can refer to a step, a `<Provider>`, or the global loading state. The global loading state is `true` when any of the other scopes are loading.

### Properties

| Name | Type | Description |
| --- | --- | --- |
| children | `(isLoading: boolean) => React.ReactNode` | A render prop function that receives `isLoading` as an argument. `isLoading` is a boolean that indicates if the current scope is loading or not. |
| scope | `string` (optional) | Specify which loading state to access. Can be a step, a provider, or the global loading state. If `<Loading>` is used outside a `<Step>`, the scope will default to "global". If used inside a `<Step>` the scope will default to the current step. For external authentication providers, the scope needs to be manually defined in the format of `provider:<provider name>`. |

Relying on the auto-inference of the scope can be helpful when using `<Loading>` in shared components, which might be referenced inside different steps. You won't need to manually pass the scope as a prop to your shared component.

### Examples

#### Basic

```tsx
<SignIn.Root>
  <Clerk.Loading>
    {(isGlobalLoading) => isGlobalLoading ? "Global Loading..." : "Global"}
  </Clerk.Loading>
  <SignIn.Step name="start">
    <Clerk.Provider name="google">
      <Clerk.Loading scope="provider:google">
        {(isLoading) => isLoading ? "Loading..." : "Continue with Google"}
      </Clerk.Loading>
    </Clerk.Provider>
    <SignIn.Action submit>
      <Clerk.Loading>
        {(isLoading) => isLoading ? "Loading..." : "Submit"}
      </Clerk.Loading>
    </SignIn.Action>
  </SignIn.Step>
</SignIn.Root>
```


#### Nested loading states

To target the loading state of a specific `<Provider>`, you need to specify the scope as `provider:<provider name>`.

```tsx
<SignIn.Root>
  <Clerk.Loading>
    {(isGlobalLoading) => (
      <SignIn.Step name="start">
        <Clerk.Provider name="google" disabled={isGlobalLoading}>
          <Clerk.Loading scope="provider:google">
            {(isLoading) => isLoading ? "Loading..." : "Continue with Google"}
          </Clerk.Loading>
        </Clerk.Provider>
        <SignIn.Action submit disabled={isGlobalLoading}>
          <Clerk.Loading>
            {(isLoading) => isLoading ? "Loading..." : "Submit"}
          </Clerk.Loading>
        </SignIn.Action>
      </SignIn.Step>
    )}
  </Clerk.Loading>
</SignIn.Root>
```
